    END POINTS:

  • /person
    • Dumps all data in the database in JSON
  • /insert
    • Inserts into the database
    • Expects data in JSON format of personID, weight, yearsTraining, name, and thier age
    • Responds if there is an errory otherwise says "it worked!"
  • /person/id
    • Updates the person with the given ID(this should be named update)
    • HTTP PATCH request
    • Expects JSON data, with weight, yearsTraining, name, and age
    • Also expects an ID from the request parameters
    • Responds with an error, otherwise is silent
  • /remove/id
    • Deletes the person with the corresponding ID from the DB
    • DELETE request
    • requires ID from request parameters
    • responds with an error if it fails

CRUD OPERATION CALLS

Post database

curl -v localhost:5000/person

Insert into database

curl -v \
--header 'Content-Type: application/json' \
--data '{"personID": 2, "weight": 103, "yearsTraining": 3, "name": "john", "age": 12}' \
http://localhost:5000/insert

Patch database

curl --header 'Content-Type: application/json' \
--request PATCH \
--data '{"weight": 150, "yearsTraining": 4, "name": "terry", "age": 16}' \
http://localhost:5000/person/1

Delete from database(first entry)

curl --request DELETE \
http://localhost:5000/person/1
